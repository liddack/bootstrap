Suporte a terceiros

Apesar de oficialmente não darmos suporte a plugins e add-ons de terceiros, podemos lhe dar alguns conselhos para evitar possíveis problemas em seus projetos.

Box-sizing

Alguns apps de terceiros, incluindo o Google Maps e o Google Custom Search Engine, entram em conflito com o bootstrap devido à * { box-sizing: border-box; }, uma regra que faz com que o padding não afete a largura final computada de um elemento. Saiba mais sobre box model e dimensionamento no Guia para Iniciantes do Tableless.

Dependendo do contexto, sobrepor essa regra conforme necessário (Opção 1) ou redefinir o box-sizing de regiões inteiras (Option 2).

{% highlight scss %} /* Resetando box-sizings * * Redefina elementos ou sobreponha regiões para evitar conflitos com as * configurações globais de box-model do Bootstrap. As duas opções, sobreposições * individuais e redefinições de regiões, estão disponíveis tanto em CSS puro * como em Less não compilado. */ /* Opção 1A: Sobrepor o box-model de um único elemento via CSS */ .element { -webkit-box-sizing: content-box; -moz-box-sizing: content-box; box-sizing: content-box; } /* Opção 1B: Sobrepor o box-model de um único elemento através de um mixin do * Less do Bootstrap */ .element { .box-sizing(content-box); } /* Opção 2A: Redefinir uma região inteira via CSS */ .reset-box-sizing, .reset-box-sizing *, .reset-box-sizing *:before, .reset-box-sizing *:after { -webkit-box-sizing: content-box; -moz-box-sizing: content-box; box-sizing: content-box; } /* Opção 2B: Redefinir uma região inteira com um mixin personalizado do Less */ .reset-box-sizing { &, *, *:before, *:after { .box-sizing(content-box); } } .element { .reset-box-sizing(); } {% endhighlight %}
